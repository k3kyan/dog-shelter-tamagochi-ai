# Dog Shelter Tamagotchi AI

A pet simulation game where you adopt a rescue dog and build trust with it over time. As you chat with the dog, its responses change depending on how much it trusts you based on its statistics. You can also ask questions about the dog and how to care for it using the care guide chat feature, which is built on a RAG pipeline.

**Live demo:** https://d3ugxz2xwrzex6.cloudfront.net/

---

## What It Does

1. **Adopter profile** — Answer a short quiz (living situation, climate, experience level). The app scores and ranks dog breeds by compatibility with your lifestyle.
2. **Shelter screen** — Browse your top breed matches and choose one to adopt.
3. **Rescue screen** — Care for your dog (feed, play, walk, rest). Stats decay over time. Trust grows as you care for it.

The dog moves through four trust stages as you play:

| Stage | Trust | What changes |
|---|---|---|
| Withdrawn | 0–29 | Scared responses |
| Cautious | 30–59 | Wary but curious |
| Warming | 60–84 | More of the breed's personality starts showing |
| Thriving | 85–100 | Fully itself — joyful, expressive |

Each breed has its own personality injected into the LLM system prompt (temperament, energy level, trainability), so a Border Collie and a Basset Hound respond differently once they trust you.

## AI Features

### RAG Pipeline (dog care Q&A)
The in-game "Care Guide" chat allows players to ask dog care questions and get breed-specific answers grounded in real articles.

- Scraped ~300 dog care articles from PetMD's website about specific breeds and general dog care
- Chunked text with LangChain's `RecursiveCharacterTextSplitter`
- Used the contextual retrieval technique by generating an LLM context summary per chunk using Groq to improve retrieval quality
- Embedded chunks with `sentence-transformers` using the all-MiniLM-L6-v2 model via HuggingFace Inference API
- Stored vectors in `Pinecone` (originally and locally uses `ChromaDB`)
- At query time: embed the player's question by finding the top 3 similar Pinecone semantic search results, which then the LLM generates a grounded answer to return along with the source URLs from the chunk metadatas.

### LangChain Statistics-Aware Chat Agent
The dog's chat responses are generated by a LangChain agent that adapts based on the current trust stage.

- Dynamic system prompt injection: breed data (temperament, personality), live stats (hunger/energy/happiness/health), and trust stage are injected per request
- Custom `@tool` created named `suggest_activity` which lets the LLM recommend a player action based on current stats. Created this so the LLM recommends a specific action rather than a generalized action.

## Project Structure

```
scraper/                article scraping (BeautifulSoup)
ragpipeline/            chunking, embedding, ChromaDB / Pinecone database creation
pipeline/               ETL — cleans and fuzzy joins dog breed info + dog shelter intake datasets
api/                    Backend API
  routes/               FastAPI route handlers
  models/               Save data to valid DynamoDB structures (Python floats to DynamoDB Decimals)
  schemas/              API request parameter validations (Pydantic BaseModel for request validation)

  services/             Helper methods to move logic elsewhere to not clutter the routing endpoints

    rag_chat_agent.py   RAG retrieval + answer generation
    trust_system.py     trust stage logic
frontend/               React app
```

## Running Locally

The `frontend` branch is set up for local development.

```bash
python -m venv pipelinevenv
pipelinevenv\Scripts\activate

python run_pipeline.py
```

**Scraper**
```bash
# No need to run since I already scraped the articles, but if need to rerun or want to scrape more articles, just do
cd scraper
python -m venv scrapervenv
scrapervenv\Scripts\activate
pip install -r requirements.txt
python url_scraper.py #if you want to scrape urls of articles from a website, but its not perfect so will need to go to the urls to make sure its an article and not a page of articles
```

## Architecture Notes

- The Lambda function downloads `breed_profiles.parquet` from S3 on cold start, making breed data available without a separate database
- Originally used ChromaDB + local sentence-transformers for the vector store, but both were too large for Lambda's deployment package limit and caused 403 errors with cold start and 10 minute CodeBuilds, so --> migrated to Pinecone + HuggingFace Inference API to resolve cold start failures by offloading these responsibilities to Pinecone and HF
- Lazy singleton pattern used for the Pinecone client so it initializes on first request (after Lambda startup), not at import time (because previously I had ChromaDB lazy singleton in attempt to help long cold start, but I switched to Pinecone and just kept a similar implementation)
